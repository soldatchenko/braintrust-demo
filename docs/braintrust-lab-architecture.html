Why this architecture works for the interview

This is essentially the Braintrust ToolRAG cookbook pattern extended with a real document source (S3), a production-grade vector store (Pinecone), and a CI/CD eval loop (GitHub Actions). It demonstrates all three Braintrust pillars — tracing, evaluation, and observability — while connecting to your existing AWS and Terraform fluency. The AI Proxy handles LLM routing and caching so every call is traced without extra instrumentation.

High-Level Build Approach

Accounts & Keys

Sign up for Braintrust free tier. Create a Pinecone serverless index (free). Grab API keys for OpenAI, Braintrust, and Pinecone. Store them in .env locally and GitHub Secrets for CI.

braintrust.dev/signup pinecone.io platform.openai.com

Document Ingestion Pipeline

Collect 20-30 markdown or text documents — Braintrust's own docs work great. Write a Python script that chunks by heading, generates embeddings via OpenAI, and upserts vectors + metadata to Pinecone. Optionally store raw docs in S3 and trigger ingestion via Lambda.

openai embeddings pinecone upsert chunking logic S3 (optional)

RAG Query Function

Build the core retrieval + generation flow. Query comes in, embed it, search Pinecone for top_k chunks, assemble a prompt with retrieved context, call GPT-4o-mini via Braintrust AI Proxy (wrap_openai). Every call is auto-traced.

braintrust.wrap_openai pinecone query prompt template auto-tracing

Build a Golden Dataset

Create 30-50 question/expected-answer pairs covering your doc corpus. Mix factual, multi-hop, and "I don't know" cases. Upload to Braintrust as a versioned dataset. Traces from Step 3 can also be promoted to test cases.

braintrust datasets QA pairs edge cases

Scoring & Evaluation

Wire up autoevals scorers: Factuality for answer correctness, ContextRelevancy and ContextPrecision for retrieval quality, Faithfulness for hallucination detection. Run via braintrust eval CLI. Compare experiments when you tweak top_k, chunking, or prompt.

autoevals.Factuality autoevals.ragas.* LLM-as-judge braintrust eval

Experiment & Iterate

Run multiple experiments varying the knobs: different chunk sizes, top_k values, prompt templates, models. Braintrust auto-compares each run, showing per-test-case regressions and improvements. Use the Playground to prototype prompt tweaks interactively.

side-by-side diffs playground score trends

CI/CD Integration (Optional Polish)

Add a GitHub Actions workflow that runs braintrust eval on every push. Fail the pipeline if scores drop below a threshold. This demonstrates the production eval loop that Braintrust promotes — and ties directly to your DevOps background.

github actions braintrust eval CLI quality gates
